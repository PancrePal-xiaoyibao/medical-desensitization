# 项目 Review 报告（病历脱敏方向）

日期：2026-04-22  
范围：仅审查当前仓库中“产品定位一致性、命名一致性、体验风险、交付风险”，不修改业务代码。

## 一、核心结论

当前仓库已经具备“病历脱敏工作台”主链路能力，但仍存在较多“小馨宝心理陪伴”历史残留。  
这些残留会在品牌认知、用户信任、导出内容、会话体验和工程标识上造成混淆，建议按优先级分批清理。
社区的基金会正式捐赠项目是小胰宝，所以名称上挂小胰宝的脱敏会好一点，品牌名称建议去掉小馨宝之类的，这个是社区的另外一个未捐赠运营项目

## 二、主要发现（按严重度）

### P0（高优先级，建议立即处理）

1. **用户可见主界面仍出现旧品牌与心理陪伴语义**  
   - 文件：`src/components/ChatInterface.tsx`、`src/components/WelcomeModal.tsx`、`src/app/page.tsx`
   - 典型问题：
     - “小馨宝现在有点累了”“我是小馨宝”“在这儿，你可以放心倾诉”等文案仍在线上路径可见
     - 快捷语“我想找人聊聊天”与病历脱敏工具定位不一致
   - 风险：
     - 用户误解产品定位（心理咨询 vs 医疗信息脱敏）
     - 医疗场景下品牌可信度下降

2. **导出内容仍携带旧品牌名**  
   - 文件：`src/components/ChatInterface.tsx`
   - 典型问题：导出文件名仍为“小馨宝对话录 - xxx.md”
   - 风险：对外流转文档品牌错误，影响交付与对外演示。

### P1（中优先级，建议本周内处理）

3. **工程内部命名仍为 `xiaoxinbao`**  
   - 文件：`package.json`、`package-lock.json`、`backend/go.mod`、`backend/cmd/server/main.go`、`.github/workflows/ci.yml`、`src/store/useChatStore.ts`
   - 典型问题：
     - npm package 名、Go module、backend 二进制产物名、持久化 storage key 仍是旧命名
   - 风险：
     - 新成员理解成本上升
     - CI 产物与项目名称不一致，影响发布规范与运维识别

4. **欢迎引导和协议入口文案偏“治愈/陪伴”语境**  
   - 文件：`src/components/WelcomeModal.tsx`、`src/components/Sidebar.tsx`
   - 典型问题：
     - “开启治愈之旅”“治愈之旅已开启”等
   - 风险：与当前“病历脱敏工具”定位冲突，容易造成监管/合规语义误读。

### P2（可排期优化）

5. **脱敏产品与聊天模块边界尚未彻底梳理**  
   - 现状：
     - 脱敏流程已具备，但聊天模块仍承载较多历史产品叙事
   - 风险：后续每次迭代都容易回流旧文案或旧交互。

6. **缺少“命名一致性/文案合规”自动检查**  
   - 现状：
     - CI 未见针对关键禁用词（如“小馨宝”）的校验
   - 风险：历史名词可能反复进入主分支。

## 三、建议任务清单（仅任务，不含代码变更）

### 任务组 A：品牌与文案统一（P0）

- 全量替换用户可见文本中的“小馨宝”为“小胰宝”（或最终确认的新品牌名）。
- 移除/改写“倾诉、治愈、陪伴”类心理咨询语义为医疗脱敏场景语义。
- 统一异常提示、按钮文案、空态引导、导出文件名。

**验收标准：**
- 全站 UI 不再出现“小馨宝”与心理陪伴语境。
- 导出文件、Toast、空态、标题文案均一致。

### 任务组 B：工程命名迁移（P1）

- `xiaoxinbao` -> `xiaoyibao` 的工程标识迁移（npm name、Go module、二进制名、storage key、CI 构建命令）。
- 评估 `zustand` storage key 迁移策略，避免历史会话数据读取异常。

**验收标准：**
- 全仓搜索无 `xiaoxinbao` 残留（允许第三方依赖锁文件里不可控项时单独标注）。
- `npm run build`、`npm run dev:backend`、CI backend job 均正常。

### 任务组 C：模块边界与信息架构（P2）

- 明确“脱敏工作台”与“AI 对话”的关系：主功能为脱敏，对话为后续动作。
- 将历史心理对话相关组件标记为 legacy，或拆分为可选模块，防止默认入口混淆。

**验收标准：**
- 新用户首次进入只看到脱敏主任务路径。
- 对话能力呈现为“后续分析步骤”，非主叙事。

### 任务组 D：质量门禁（P2）

- 增加命名一致性检查（如 CI 中扫描禁用词：`小馨宝` / `xiaoxinbao`）。
- 增加最小化 UI 文案回归检查（关键页面快照或文本断言）。

**验收标准：**
- PR 阶段即可阻断旧命名回流。
- 关键页面文案回归可自动化发现。

## 四、建议执行顺序

1. 先做任务组 A（对外感知风险最高）。  
2. 再做任务组 B（工程一致性与长期维护）。  
3. 同步做任务组 D（防回归）。  
4. 最后推进任务组 C（结构优化）。
